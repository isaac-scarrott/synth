# 2026-07-07

## ⌘K rationalised — one vocabulary, five laws for every action

The command palette had grown organically: the same tier was called both "branch" and
"worktree" mid-flow, actions repeated the target name the crumb already showed ("Rename
feat/command-palette…" under a "feat/command-palette" crumb), destructive pickers led navigation
lists, and the ellipsis/verb choices were ad hoc. The surface still worked, but it read as many
hands rather than one. This pass makes every label, order, and affordance fall out of a small set
of rules, so actions are thoughtless and contextual.

**Vocabulary — one noun per concept, aligned with the rest of the app.**
- You navigate, rename, and detach by **branch** (what a dev thinks in). The word **worktree**
  appears *only* where the on-disk folder is literally the subject: **Create worktree…** (matches
  the existing kebab + "Create worktree" dialog) and **Delete worktree** (the destructive fork
  that removes the folder from disk). This is exactly ADR-0007's "every branch row is a worktree
  folder" made legible rather than leaked.
- Workspace create is **Add workspace…** (matches the "Add workspace" dialog and the `+` button).
- Sessions stay **New terminal / New Claude Code / New browser**.

**Remove ≠ Delete stays deliberate (ADR-0007) and is now spelled out.** Remove = reversible,
UI-only, nothing on disk changes; Delete = destroys (a session's process, or a worktree's folder).
Every confirm's placeholder states the consequence in plain words — "Removes it from the sidebar —
the repo and everything on disk stay", "Ends this session and its process", and the worktree fork
carries per-option tags ("removes the folder from disk" vs "folder and branch stay on disk"). The
red/plain colour + the wording carry the signal; the danger path is never one keystroke away
(destructive actions always route through a confirm frame).

**The five laws every frame now obeys:**
1. **Vocabulary** — as above; no stray "New worktree…" / "New workspace…".
2. **Remove vs Delete** — reversible vs destructive, made legible in every confirm.
3. **Ellipsis** — `…` iff the action needs you to type (Rename…, Create worktree…, Add
   workspace…, Go to address…) or opens Settings (Settings…). Immediate actions and confirm verbs
   (Remove, Delete, New terminal, Toggle sidebar) carry none.
4. **Bare naming** — inside a drilled frame the crumb already names the target, so labels are bare
   ("Rename…", "Remove", "Delete", "Settings…"), never "Rename synth…". In root *search* results,
   context actions collapse under one "Actions" group with the target as a {ctx chip}, not inline.
5. **Order** — create → navigate/list → modify (rename / settings / move / detach) → destroy
   (danger, always last).

**Simplifications that fell out of the laws:** the nav category frames (Branches, Sessions) are
now pure navigation lists — no destructive item at the top of a list you opened to jump through.
The bulk delete-pickers ("Remove workspace…", "Delete session…") are gone: every row already
carries its own Remove/Delete via its frame, so the pickers were a second, danger-first path to
maintain. Workspaces keeps only the unambiguous global create (Add workspace…). The now-unused
`deletePicker` frame was removed.

Landed in both `working.html` and `big-picture-design.html` (subset invariant preserved — the diff
is still only the title + big-picture's extra session rows). Verified live end-to-end: every frame
renders correct labels/order, and rename / create / delete-session / remove-branch (both forks) /
detach + re-adopt / `a`-key add / keyboard nav all mutate the tree as expected, in light and dark.

## ⌘K drops the ellipsis (refines the five-laws entry above)

Law 3 ("ellipsis iff you must type or it opens Settings") is retired. In a keyboard-first palette
the `…` is weak signal that reads as clutter — pressing Enter already reveals the next frame
instantly, so the dots tell you nothing you won't see a beat later. Raycast, Linear's ⌘K, and
VS Code's palette all drop them for this reason. Every action label is now a plain verb
(`Rename`, `Create worktree`, `Add workspace`, `Go to address`, `Settings`). The only trailing
`…` left is the empty-field guide "Type a branch name to check out or create…", which is a
sentence prompt, not a command.

## ⌘K, hardened from a fresh-eyes focus group (11 personas)

Eleven blind personas drove the live palette; their convergent findings drove two rounds of fixes.

**Defects fixed (A–E):**
- **Ranking** — an exact/prefix/acronym match on a row's name now dominates loose subsequence hits and
  actions-on-that-thing, and the group with the best match leads, so the top row is the strongest hit.
  (Five voices independently hit `ds`→"Add workspace" / `main`→"New terminal" / `clau`→"Rename".)
- **Fresh open highlights nothing** — a reflexive ⌘K+Enter is now a no-op, not a rename.
- **Danger stays red** — active "Delete" no longer inherits the blue is-active label in dark mode.
- **Destructive fork defaults to safe** — the branch-remove fork lands the cursor on "Remove from
  sidebar", never "Delete worktree".
- **Consequence copy is a visible wrapping line** (`.cmdk__note`), not a truncating input placeholder.
- **Esc pops one frame**, closing only at the root (matches Backspace); ⌘K still closes outright.

**Enhancements (from the panel + user calls):**
- **Recents** — the cold palette leads with a "Recent" group of your most-used STABLE targets
  (branches, workspaces, named terminals). Actions keep fixed positions; dynamically-named sessions
  (AI titles, browser URLs) are excluded. In-memory + seeded here; the native app would persist it.
- **"New branch"** — the create action reads "New branch" (a junior bounced off "worktree"); the word
  "worktree" survives only on the on-disk delete path, keeping disk-truth for the SRE.
- **⌘T** spawns a terminal in the current branch (matches the native app); added to the ⌘? sheet.
- **Ownership in ⌘K** — a focused browser's Session group now carries Detach / Move-under inline, so
  the ownership relationship isn't kebab-only.

Deliberately deferred: full screen-reader ARIA + focus-trap (the "focus-trapped input" copy currently
over-promises), a "Switch to branch" jump, and agent-name disambiguation. Both designs; invariant held.

## ⌘K, final polish from the focus-group follow-ups

Resolving the panel's remaining flags, kept deliberately lean and understated:
- **Focus trap + restore** — Tab is trapped inside the open palette (`aria-modal`), and focus returns to
  wherever you opened from on close. The "Focus-trapped input" copy is now true.
- **Reversible vs destructive, by colour** — reversible **Remove** (workspace, branch-from-sidebar) drops
  the danger-red and trash icon for a quiet minus; red + trash are reserved for real destruction (Delete
  session, Delete worktree). Red now always means "this destroys something."
- **Branch ref survives** — the worktree-remove note now says "The git branch itself stays either way."
- **Base + disk path at create** — cutting a new branch shows one muted line: the base it forks from and
  where the worktree lands on disk (e.g. "New branch off feat/command-palette · …/Synth/worktrees/synth/…").
  Transparency, not a picker — no bloat.
- **Dead menu code deleted** — the retired popover (`openMenu` / `confirmMarkup` / `showConfirm` /
  `positionMenu`), the standalone `openWorktreeDialog`, their event handlers, and the orphaned `.menu`
  CSS are gone (~11 KB); the kebab already routes through ⌘K, and the live comment card keeps `.menu__btn`.

Deferred by choice: full screen-reader ARIA (listbox/option roles), a "Switch to branch" jump, and
agent-name disambiguation. Both designs; invariant held throughout.

## Native app: this session's ⌘K work ported (SwiftUI, verified)

Everything above landed in the native app (`app/Sources/Synth/`), on the real `@Observable` store:
- **Vocabulary + verbs** — "New branch" (worktree stays only on the disk-delete path), "Add workspace",
  bare drilled verbs (crumb names the target), no ellipsis.
- **Remove ≠ Delete** — reversible Remove is a neutral minus-circle; Delete (session) and "Delete worktree"
  are trash + danger red. Bulk delete-pickers removed (browse frames are pure navigation).
- **Ranking** — exact/prefix boost, ctx as a weak fallback, groups ordered by best match; a typed name
  tops its own actions and becomes the default row (verified over the control socket: "main"→branch #1,
  "migrate"→jump).
- **Fresh open highlights nothing** (`activeIndex = -1`; reflexive ⌘K+Enter is a no-op — verified).
- **Consequence as a wrapping note line**; branch-remove fork defaults to "Remove from sidebar" and says
  the git branch survives.
- **Esc pops one frame**; **Recent** frecency group leads the cold palette (in-memory; verified); base
  branch shown on a new-branch create; **⌘T** documented in the ⌘? sheet.
- **Dead popover-menu deleted** — `RowMenu`/`Menu`/`activeMenu`/`CreateWorktreeSheet` machinery gone
  (~500 lines net removed); the kebab already routes into the ⌘K palette.

Ported via isolated worktree + sequential sub-agents (Palette.swift is coupled), each `swift build`-clean;
behavior verified headlessly over the control socket (screenshots TCC-blocked). Fidelity audit: 13/14
targets matched, 5 small gaps fixed. Known deltas from the design, deliberately deferred: recents are
session-only (design also floats branches/workspaces), and the new-branch line shows the base but not the
on-disk worktree path.
