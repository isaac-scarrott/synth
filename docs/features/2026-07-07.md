# 2026-07-07

## In-app feedback (⌘⇧F) — one textbox that forks: author spawns a fix, everyone else emails

Feedback on Synth, from inside Synth, with the least possible friction: one textbox, opened from
anywhere with **⌘⇧F** (also a ⌘K "Send feedback…" action and a row in the ⌘? sheet). Enter is a
newline, **⌘↵ sends**, Esc dismisses; an unsent draft persists across reopens. The surface is
identical for everyone — the fork lives only in the outcome, decided silently at send.

The insight that makes it cheap: **this is browser comment-mode pointed at Synth itself.** The
author path reuses the exact machinery that already ships — lazy worktrees, `spawnClaude`, and
CommentMode's boot-and-wait PTY delivery — so the novel code is glue, not new primitives.

- **Two modes, one surface, resolved once at launch (`FeedbackMode`).** Author when git
  `user.email` matches a known author address (or `SYNTH_AUTHOR=1`); everyone else emails
  (`SYNTH_AUTHOR=0` forces it). The user never picks a mode and, until send, never sees a
  difference.
- **Author → a real `feedback/<slug>` worktree with Claude already on it.** `submitFeedback`
  slugifies the first four words, cuts a `feedback/<slug>` branch off the open session's workspace
  (`createWorktree`'s path, with a new `onReady` hook on `materialize` so it spawns a single
  seeded Claude instead of the full session template), and — once the checkout lands and the hook
  seam confirms the Claude live — pastes the feedback + captured context into its PTY. The seed is
  delivered **only to a hook-confirmed-live Claude, never a bare shell** (CommentMode's security
  invariant, carried over verbatim). The session appears quietly; an "On it · feedback/<slug>"
  toast confirms.
- **Everyone else → a pre-filled `mailto:isaac.scarrott11@gmail.com`.** Subject + body built with
  `URLComponents` (so `&`/`+`/newlines encode correctly), capped at ~1600 chars for mailto's
  practical limit. No mail client → the body is copied to the clipboard and a toast says so.
- **Context is captured silently, structural only — never PII.** The author seed carries
  `Here: <session kind> · <branch>`, `State: N workspaces · N working · N unread`, and
  `Env: Synth <version> · macOS <ver> · <theme> theme`. Never file contents, absolute paths,
  terminal output, env values, or the clipboard. The **email body attaches only version/OS** — no
  branch or session names leave the machine. There is no context UI (an earlier "secrets stripped"
  chip was cut for simplicity): the capture is an allowlist of scalar facts, so nothing outside it
  can leak.
- **Design first, then ported.** Landed in `working.html` + `big-picture-design.html` (diff stays
  title + demo rows only), then into the native app: new `FeedbackMode` + feedback methods on
  `AppStore`, `FeedbackSheet` (mirrors `CreateWorktreeSheet`/`DialogFrame`), the ⌘⇧F branch +
  modal-Esc guard entry + overlay in `SynthApp`, the two palette items, the shortcuts row, and
  `GitService.gitUserEmail()`.

Verified: the design drives correctly in a browser (⌘⇧F opens, author send spawns a quiet working
`feedback/*` branch without stealing focus, guest send builds the mailto). The native app builds
clean; the author path was driven headlessly over the control socket (TCC denies synthetic input
on this machine) — it captured structural-only context, ran a real `git worktree add -b
feedback/sidebar-lag-when-i`, spawned a Claude session into it, and raised the "On it" toast; the
mailto URL was confirmed well-formed (`&`→`%26`, newlines/unicode encoded, round-trips). An
independent fidelity audit against the design surfaced two gaps since fixed (the `· N unread`
count, and ASCII-only slugs).
