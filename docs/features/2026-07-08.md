## 2026-07-08 — Split / monitor: the content pane can tile 2+ linked sessions (both designs)

The content pane has always shown exactly one session — a pure function of the single open session.
This adds a **split / monitor** view: 2 or more sibling sessions tiled side by side, so you can watch
the app in an embedded browser while its dev-server logs stream beside it, ± the Claude agent driving
them. Explored end-to-end first in a standalone prototype (`split-explorations.html`), stress-tested
by a focus group + a jedi-council, then landed in both canonical designs. Native SwiftUI port is a
separate, later effort.

**The model.**

- **A split is a persistent group, not transient view state.** `LINKS` holds arrays of session rows.
  Navigating away (opening a session outside the split) drops you to that session **solo**, but the
  split lives on in the sidebar and reopens from its capsule. Opening any member reopens the whole
  group, focused on that member.
- **The sidebar shows a linked group as an Arc-style inline capsule of pills** — side by side,
  **mirroring the panes on screen** (left→right pills = left→right panes). The focused pane's pill is
  raised and keeps its name; at 3+ pills the inactive ones collapse to icon-only. A subtle grey
  trough, no box, no blue. The member rows fold into the capsule (hidden) and return to normal rows
  when unlinked.
- **Same-level rule:** you can only link **siblings** — sessions in the same worktree's list. Enforced
  silently (the picker only offers siblings; drag/keyboard paths re-check). This deliberately rejects
  cross-worktree monitoring, siding with Synth's ownership tree.
- **Focus + zoom.** The focused pane is full-colour with a blue ring; the others recede. `⌘↵` zooms
  the focused pane full (and is the novice's way back); a floating "Zoomed — back to all" chip and
  `esc` return.

**The interactions (this is where the parked questions were resolved).**

- **Adding to a split is keyboard / drag / picker — never a plain click.** `⌘\` opens *Open beside* —
  **built as a `⌘K` palette frame, not a bespoke dialog** (sharing that primitive: same list rows,
  same fuzzy filter, same ↑/↓·↵·esc·back·focus-trap). It lists the focused pane's siblings (not
  already tiled) plus New terminal / Claude Code / browser here; choosing one tiles it beside. New
  sessions are spawned through the same `addSession` primitive the palette's other create actions use.
  Dragging a session row over the panes also drops it into the split; clicking a session only ever
  *opens* it.
- **Every add appends at the end** — not beside focus, not at the exact drop position. One rule, and
  the newly added pane takes focus. (Isaac's call over the two alternatives I proposed.)
- **No column ceiling** — the old linear cap of 3 is gone; keep adding and the panes share the width.
  Left to the user's judgement rather than a hard limit.
- **`⌘B` (collapse sidebar) hands its ~260px back to the tiled panes** — a natural focus-mode for
  watching 2–3 sessions. Falls out of the existing grid collapse for free.
- **No owner↔browser join in the capsule (for now).** The prototype joined an owned browser to its
  Claude owner with a link glyph, which forced the capsule to reorder and stop mirroring the screen.
  Dropped: the capsule is now a straight mirror of pane order. Ownership still shows in the tree (the
  browser's sibling row keeps its link chip) — just not inside the capsule.
- **Keyboard verbs:** `⌘\` open beside · `⌘⌥←/→` (or `⌘⌥H/L`) move focus · `⌘↵` zoom · `⌘W` unlink the
  focused pane (the group persists, dissolving only when it drops to one) · `esc` collapse the tiled
  view to the focused pane. Panes carry per-pane hover controls (zoom / unlink) and a draggable seam
  to resize.

**How it landed in the designs.** Single-session render is unchanged — one open session is still a
bare `.pane`, byte-for-byte as before; the split structure (`​.panes > .spane`, the capsule, the
picker) is purely additive and only appears at 2+ panes. `openEl` now aliases the focused pane, so all
the existing single-session machinery (the active-branch pill, notifications, browser page verbs)
keeps working against the focused pane. Both files ship a **seeded 2-pane default on load** — the
Claude agent beside the app it's driving (`localhost:5173`) — so the monitor view is visible at a
glance; removing the one-line `LINKS` seed lands on a single session instead.

Incidental fix found while wiring this up: `working.html` (and its twin) carried a **duplicate
`const ICON_EXTERNAL`** at IIFE scope — a fatal `SyntaxError` that was breaking the entire script.
De-duplicated to the single earlier declaration; both files now parse and run clean.

The split keybindings sit in a capture-phase listener so they only win the keys that matter while a
split is live, and defer to everything that legitimately owns the keyboard. The listener bails
whenever a palette, picker, row menu, shortcuts sheet, **any modal dialog (`.backdrop`), Settings
mode, or a focused text field** is up — so `⌘↵` submits the feedback sheet / sends a browser comment
instead of zooming a pane behind it, `⌘W`/`⌘⌥←→` never mutate a split hidden under a dialog or while
you type an inline rename, and Settings is completely inert to the split keys and restores the tiled
view on exit. Where it does act: `⌘↵` yields to a pending notification jump (its standing meaning)
and only zooms when the deck is empty; `Esc` unzooms from anywhere but collapses the tiled view only
when focus is in the panes, so `Esc` from the sidebar still hands focus to the content as before. The
pre-existing `⌘↵` notification-jump handler also grew the same overlay guard, so it can't fire behind
an open palette/dialog. Nav resume learned about the capsule too: with the open session folded into a
pill, the keyboard cursor resumes from that split's branch group instead of jumping to the tree top.

Verified by a full keyboard + click regression in a real browser (light + dark): seed renders; `⌘\`
picker appends a 3rd/4th pane past the old cap; drag-into-panes splits; focus-move, `⌘↵` zoom, `⌘W`
unlink, unlink-to-dissolve (capsule folds back to rows) all work; navigating away keeps the capsule
and reopening from a pill restores the split; `⌘B` reclaims 260px. Every pre-existing interaction
re-checked green: sidebar nav (`j/k/h/l`, Enter, Tab), `⌘K` palette, `⌘,` settings (round-trip
preserves the split), `⌘0/⌘1` focus, `⌘/` shortcuts, `⌘⇧F`, browser page verbs, `a/r/d`, the kebab
→ palette, sidebar drag-reorder, notifications (`⌥N` + `⌘↩` jump), and workspace/branch toggles.
`diff working.html big-picture-design.html` stays title + the two demo session rows only.
