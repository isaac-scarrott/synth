# 2026-07-11

## The browser MCP server records video (`browser_record_start` / `browser_record_stop`)

An agent can now record a Synth browser session as a video: start, drive the page with the
existing tools, stop — and get back a file path plus duration/frame stats. The capture is
Chromium's `Page.startScreencast` over the same CDP connection everything else uses: a JPEG per
repaint with a timestamp, ack'd frame-by-frame, spilled to a temp dir (capped at 4500 frames,
~3 minutes of continuous repaints) so a long recording can't blow the server's memory. A spike
settled the one real unknown first: CEF's CDP endpoint does emit screencast frames, and keeps
emitting them across cross-page navigations — the recording survives a `browser_navigate`.

Encoding replays the variable-rate frames onto a constant 25fps timeline (each tick repeats the
latest frame at or before it — Playwright's own screencast-to-video strategy) and pipes them
through whatever ffmpeg is available: a full build (PATH/Homebrew) yields H.264 mp4; otherwise
Playwright's ~2MB VP8-only build from its browser cache — downloaded on demand via
`playwright-core cli.js install ffmpeg` — yields webm. `image2pipe` is the one input method the
minimal build supports (no concat demuxer), which is why the constant-fps pipe is the single code
path for both. Zero new dependencies. An explicit output path picks the format by extension; an
`.mp4` request without a full ffmpeg errors with the `brew install ffmpeg` remedy instead of
silently switching containers.

The tool result is a path, not the video: MCP tool results carry text and images only, and the
model can't watch video anyway — the recording is for the user's eyes or post-processing (the
landing page's script-driven shots being the motivating case). The stop tool's description says
so, and points agents at `browser_screenshot` for checking state.

**Verified.** Driven end-to-end through a real MCP client against the live app: record → navigate
→ scroll → stop produced a playable 1600px H.264 mp4 whose extracted mid-frame shows the scrolled
page; a second recording on the same session wrote a VP8 webm to an explicit path; the minimal
build's exact arg-list was proven separately against Playwright's ffmpeg binary. Error paths
exercised: stop-before-start, double-start, and (by design) zero-frames — a page that never
repaints is an error, not an empty file.
