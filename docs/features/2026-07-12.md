# 2026-07-12

## ⌘N opens the new-session picker for the branch you're in

`⌘N` now does app-wide what `a` does on a sidebar row: it drops straight into the ⌘K
"New session" frame — terminal, each coding agent, browser — for the branch you're in.
Context resolves exactly like ⌘T's: the focused sidebar row's branch when the keyboard owns
the sidebar, else the open session's branch, else the first available. Where ⌘T commits to
the most common choice in one keystroke, ⌘N is the same intent one step earlier — "new
something here" — without reaching for the sidebar first.

Natively the binding lives in the File menu (`New Session…`), *replacing* the stock
"New Window" ⌘N rather than sitting beside it: Synth is a one-window app, so the default
binding was dead weight on the app's most conventional "make a new thing" chord. Menu
key-equivalents resolve before the first responder sees the event, so it works over a
focused terminal or browser page, same as ⌘T. Both shortcuts sheets (⌘?) list it.
