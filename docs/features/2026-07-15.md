# 2026-07-15

## PR indicators: a branch's pull request, in the sidebar and the header

Every branch row now carries the state of its GitHub pull request, and the open session's
header carries a clickable chip through to it.

**Sidebar** — a small glyph rides beside the branch name (identity, so it stays clear of the
right-hand roll-up axis reserved for session status). Colour *is* the state: green open,
purple merged, red closed — GitHub's own hues, tuned per theme. Open and closed both wear the
git-pull-request glyph (colour tells them apart); merged wears git-merge.

**Header** — the open branch's PR shows in the pane header's top-right as a raised button, like
every other control, so it reads as clickable. The state colour lives only in the git glyph +
`#<number>`, not a full fill — a control, not a status lamp. (Two earlier cuts were wrong: a
filled state-colour pill read as a loud status badge, and a bare no-fill link didn't read as
clickable at all; both from feedback.) Clicking opens the PR in the user's **default** browser
(`NSWorkspace.open`), not Synth's own — a PR page is the real browser's job, where the login
lives. On the dev channel it shifts left 64px to clear the 54px DEV badge that floats in the same
corner (`.app.is-dev` in the mock; `isDevChannel` trailing pad natively).

**Data** — `PRService` reads `gh pr list --state all --json number,state,url,headRefName` in the
workspace's repo directory (so `gh` infers the repo from its remote — no owner/name to hardcode).
One PR per head branch, strongest kept (open > merged > closed, then most recent). A missing or
unauthenticated `gh`, or a repo with no GitHub remote, all resolve to "no PRs" — never an error.
State is derived, not persisted (like session status): fetched off the main thread at launch, on
add-workspace, and on every app activation, so a branch that merged while the user was away
updates on their return.

**Number formatting gotcha** — SwiftUI `Text("#\(Int)")` is a `LocalizedStringKey` and groups
the digits (`#13,874`); a PR number is an identifier, not a quantity, so both the chip and the
tooltips use `Text(verbatim:)`. Caught by driving the real app against `cli/cli`'s live PRs.

Verified end to end in the running app against real GitHub data: all three states render with the
right glyph and colour in the sidebar, and the header chip opens the PR. Both designs (subset
invariant held) + native.

## Copy the branch name from the pane header

A copy button sits just after the `workspace / branch` crumb in the pane header — one click puts
the branch name on the clipboard and flashes a green check. Hover-revealed on the header (the
sidebar-kebab idiom), so it stays out of the way until wanted. It's a sibling of the crumb, not
inside it, so a long branch name that ellipsizes never clips the button. Mock copies via
`navigator.clipboard`; native via `NSPasteboard`. Both designs (subset invariant held) + native;
the hover-reveal was confirmed by force-showing it under automation (the headless screenshot can't
hover), then reverting.

## Synth 0.4.0 — everything since 0.3.0, shipped

0.4.0 (build 212) is a minor: it rolls up the run of feature work that landed after 0.3.0 —
native browser device mode (⌘⇧M) with an agent-drivable `browser_device_mode` tool, PR indicators
in the sidebar and pane header, the copy-branch-name button, native notifications from the open
session too, the sidebar restyle, and the agent-worktree approval moving into ⌘K — plus the
process-lifecycle and memory work (session-tree reaping, crash-orphan reaper, MCP servers exiting
on parent death, bounded leaks closed) that has no visible surface but is the reason a long-running
Synth stays honest about what it's holding.

Delta economics held again, and better: nine deltas at 6.9MB total against a 136MB full download,
because `app/releases/` still carried 171/181/185/191, so `generate_appcast` could diff 212 against
every build a user might be upgrading from. An installed copy on any of those four gets a
sub-MB-to-~1MB update.

Verified as an outsider would see it — a credential-less download with the quarantine bit set:
`spctl` reports `accepted / Notarized Developer ID`, `stapler validate` passes offline, the bundle
reports 0.4.0 / 212, and every appcast enclosure (the full zip plus all four deltas) carries an
`edSignature`. Installed to `/Applications` from the stapled build.
