## Anonymous product analytics (PostHog)

Synth now has opt-out, anonymous usage analytics so we can see how many people run it, which
features they use, how often, and whether they stick Рђћ without collecting anything that identifies
anyone. The SDK is PostHog's `posthog-ios` (`from: 3.59.3`, resolves 3.66.x; it ships macOS
support), added as an SPM dependency alongside Sparkle.

**One seam.** `Analytics.swift` is the only file that imports `PostHog`; the rest of the app calls
`Analytics.capture(_:)`, `Analytics.error(_:)`, `Analytics.isEnabled(_:)`, `Analytics.setOptOut(_:)`.
It's a `@MainActor enum` with no state beyond a `live` flag, so misuse before bootstrap is a silent
no-op rather than a crash.

**Anonymous by construction.** No `identify()` is ever called, so every install is a random
per-machine id Рђћ no account, no login. Config: `personProfiles = .identifiedOnly`,
`captureApplicationLifecycleEvents = true` (this is what feeds retention/stickiness),
`captureScreenViews = false`. Session replay, element-interaction capture, and surveys are iOS-only
in the SDK and simply don't exist on macOS, so the screen is never recorded and no UI content leaves
the machine. Two registered super-properties: `channel` (dev/stable) and `app_version`.

**Three gates keep the numbers clean and the default safe.** Analytics is inert unless all hold:
not the dev channel (so the author's own runs never skew usage), the compiled-in `projectKey` is a
real `phc_Рђд` token (so CI and forked checkouts stay silent Рђћ it ships as a placeholder until the
Synth PostHog project exists), and the user hasn't opted out. The project token is a *publishable*
client key, not a secret, so it's fine in source and in the shipped binary.

**Opt-out.** `AppStore.analyticsEnabled` (UserDefaults `synth-analytics-enabled`, default on) drives
a **Settings Рєњ Privacy** toggle built from the same `mcpRow` component as the MCP/sound toggles.
Flipping it calls `PostHog.optOut()/optIn()` immediately and persists; `Analytics.bootstrap` reads
the same key straight from defaults at launch (via `AppStore.loadBoolPref`, before the store is
alive) so the *first* event already respects the choice. EU data region (`eu.i.posthog.com`).

**First event set** (deliberately small, all non-PII):
- `session_created` Рђћ `{ kind, agent_initiated }`, from the single `addSession` chokepoint every
  terminal/browser/agent session routes through.
- `worktree_created` Рђћ `{ from_template }`, from `materialize`'s `.ready` case.
- `feedback_submitted` Рђћ `{ mode, has_body, length }` from `submitFeedback`. Records *that* feedback
  happened and roughly how big, never the text Рђћ the content still reaches the author through the
  existing email / fix-agent paths.
- app opened/closed Рђћ free, via PostHog's lifecycle events.

**Settings-pane parity note.** The Privacy toggle lives only in the native app. `working.html` has no
Settings-detail mock (the sound and MCP toggles are already app-only), so the working Ріє big-picture
subset invariant is untouched Рђћ nothing to mirror.

**Not done yet (follow-ups).** Native crashes Рђћ signals, `fatalError`, the vendored Ghostty/CEF C++
engines Рђћ are *not* captured; `Analytics.error(_:)` only covers caught Swift errors. Real crash
capture needs a dedicated handler (a signal handler or MetricKit's `MXCrashDiagnostic`). Feature
flags are wired (`Analytics.isEnabled`) but nothing gates on one yet. And the analytics stays a
placeholder no-op until the "Synth" PostHog project is created and its key pasted into
`Analytics.projectKey`.
