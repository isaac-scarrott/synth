# 2026-07-23

## Experimental Tabs — a two-deep sidebar with a tab strip per pane (off by default)

An opt-in view mode that re-presents the existing content model as tabs, decided in a grill-with-docs
session and recorded in ADR-0014's 2026-07-23 amendment. It ships behind one global Settings toggle,
**off by default**, and is presentation-only: the store stays `branch → pane-tree → sessions` whether
the flag is on or off, so it flips instantly and losslessly with no migration. Flip it off and the
sessions are back in the tree.

**A tab is a session's handle, nothing more.** The glossary forbids *calling* a session a tab, and
that holds — the tab is the clickable header that carries a session's name, icon, and its
live/unread/needs-input signal. `CONTEXT.md` now carries the **Tab** entry (experimental) and the
**Session** / **Belongs to** notes that keep it honest. No new entity, no new persisted structure.

**Two things move when the flag is on.** The sidebar drops to **two deep** — sessions leave the tree,
the branch becomes the deepest row carrying only its roll-up, and every per-session signal rides on
the tab instead. And each pane draws a **strip of tabs**, one per session it hosts, one active. The
strip shows even for a lone tab (so the `+` new-tab affordance and the drop target never vanish), is
uncapped, and overflows by scrolling.

**Tabs live inside the ADR-0014 pane tree; splitting stays.** A leaf now binds a strip of ≥1
sessions, one active — the ADR's *"one session per pane"* invariant amended to *"a strip of ≥1, one
active."* A session is a live surface and can live in only one strip at a time, so **splitting moves
a tab, it never duplicates it** — the divergence from Cursor/VSCode. `⌘⇧arrow` *sends* the active tab
toward the arrow: into the neighbour pane if one exists there, else a new split.

**Nothing is pinned** — the explicit rejection of Cursor's fixed chat rail. The agent is a peer tab,
orderable and closable like any other. An agent-opened browser lands as a tab in its **owner's** pane
strip, wearing the owner-mark, with an unread dot and no focus-steal.

**Lifecycle.** `⌘N`/`⌘T` open a new tab in the active pane. Closing the last tab in a pane collapses
it and reflows the sibling (`pruneLayout`); closing the last tab of the last pane drops the branch to
**dormant** (worktree kept) — the zero-session case the taxonomy already names. Switching branches
swaps the whole tab set (layout is per-branch); an attention-jump activates the tab that raised it.

**Keyboard.** Panes keep everything they had — `⌘1–9` focus, `⌘⌥arrows`/`⌘⌥HJKL` move focus,
`⌘⇧↩` zoom, `⌘⇧U` unsplit, `⌘`` cycle. Tabs add `⌘⇧[`/`⌘⇧]` switch and `⌃⇥`/`⌃⇧⇥` cycle; `⌘W`
closes the active tab (`⌘D` stays a silent alias so tabs-off muscle memory survives); `⌘⇧arrow`
sends a tab toward the arrow. Reordering within a strip is mouse-drag (rare, fiddly, not worth a
chord).

*Rejected:* a separate persisted tab-group model (would make the toggle a migration and forfeit the
lossless flip); a per-tab number chord (an uncapped strip can't earn muscle memory — which tab is 6?
— so numbers stay on the few, spatial panes); refusing to close the last tab; hiding the strip until
a second tab exists. Lands first in both design files behind the toggle, then the native port amends
the `Layout` invariant rather than forking a second layout path.
