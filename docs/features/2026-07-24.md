# 2026-07-24

## Experimental Tabs — the native port lands (off by default)

The single-strip Tabs view mode decided in the 2026-07-23 mock (ADR-0014's 2026-07-23 amendment,
`CONTEXT.md` **Tab**) is now implemented in the native SwiftUI app, gated behind the same
off-by-default global preference so a tabs-off build is byte-for-byte today's behaviour. It is a
faithful port of `working.html`, not a redesign — the store stays `branch → pane-tree → sessions`,
and every tabs-gated path keys off `AppStore.tabsMode`, so flipping the toggle re-renders the same
store losslessly with no migration.

**The flag.** `AppStore.tabsMode` — a persisted global preference mirroring the theme/sound prefs
(`UserDefaults`, key `synth-tabs`), off by default. An `@Observable` read, so the sidebar and content
re-render the instant it flips. Settings gains an **Experimental → Off/Tabs** segmented control
(global scope only, mirroring the Appearance control).

**Two-deep sidebar.** With tabs on, `BranchRow` drops its disclosure chevron, keeps its Reveal shut
(session rows + the split echo band leave the tree), and forces the branch roll-up visible even while
nominally expanded — the branch's only per-session signal. `peeking`/`isActivePill` are made
tabs-aware so the branch keeps its "you are here" pill when its sessions no longer show below it.

**One strip per branch (`TabStrip.swift`).** The horizontal twin of the sidebar rows: the branch's
sessions in order, the on-screen split's members bonded into a contiguous cluster at their first
member's position (styled like the sidebar's split band — rounded chips on `raised`, active member
accent-ringed), the rest lone tabs, then `+` and a right-aligned relocated PR chip. Selecting a lone
tab full-screens it (the durable split stays shown as the bonded cluster, stashed behind the
full-screen); selecting a member returns to the split. The per-pane header (`PaneHead`) is hidden in
tabs mode; when the sidebar is collapsed the strip carries the expand toggle past the traffic lights.
Reuses the sidebar's exact indicator chrome (`SessionIcon`/`StatusIndicator`/`OwnedIndicator`) — an
agent-owned browser wears its owner-mark; unread lands as a blue dot on the tab icon.

**Keyboard.** `⌘⇧[`/`⌘⇧]` and `⌃⇥`/`⌃⇧⇥` step the branch's tabs (`cycleTab`), `⌘1–9` selects the Nth
tab (`focusPane` tabs-aware), `⌘W` closes the active tab through the same confirm path as `⌘D`. The
⌘? shortcuts sheet inserts a **Tabs** category after General and rewrites the Split-layout rows tabs
changes (⌘⇧arrow becomes "Send tab", ⌘⇧U becomes "Merge pane"), so the sheet always matches reality.

**Drag.** A tab's unified drag mirrors the sidebar's `SessionRowDrag` on the horizontal strip:
over a pane → split it at the pointer; squarely over another tab → pair the two into a split; between
tabs → reorder the branch's sessions (horizontal twin of the reorder line); a split member dragged
back to the strip leaves its split. Reuses the same store ops (`dropZone`/`performDrop`,
`pairTarget`/`performPair`, `unsplitSession`) — which already enforce "a session lives in one leaf,
never duplicated" — plus a new horizontal `reorderTab`/`tabReorderLine`.

All slices runtime-verified by driving the built app (split-with-cluster, lone-tab-with-stashed-split,
settings toggle, `⌃⇥` cycling, the shortcuts Tabs group) and the tabs-off default confirmed unchanged.
