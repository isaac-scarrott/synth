# 2026-07-24

- **⌘K reliably focuses the palette input again** — opening the palette set focus on its
  SwiftUI `TextField` via `@FocusState` in `.onAppear`. Making any text field first responder
  triggers AppKit's password-autofill heuristic, which asks SwiftUI to gather every focusable
  key-view in the window (`_recursiveGatherAllKeyViewCandidates` → `FocusNavigator.allItems`), a
  main-thread traversal with an uncached protocol-conformance lookup per node. `sample` put it at
  ~240ms per open, and it scales with the view tree — so a long-running session that had piled up
  sessions and panes widened the window during which the `@FocusState` set silently never applied,
  leaving the input unfocused and dropping the keystrokes into the pane underneath. That regression
  grew as the app got more loaded, which is why it "used to work." Fixed by bridging the input from
  AppKit (`PaletteQueryField`, an `NSViewRepresentable`) and grabbing first responder imperatively
  with `makeFirstResponder` — immune to the starved SwiftUI focus cycle. Live filtering, the
  dash-spaces transform, frame-drill refocus, and rename-seed select-all are preserved. Driven-app
  verified across empty and heavily-loaded trees. Left standing: the ~240ms key-view traversal
  itself is inherent to SwiftUI computing the window's key-view loop on focus; killing it would mean
  isolating the palette in its own window or pruning focusable candidates — a larger change, not
  bundled here.

- **Synth 0.10.0 shipped (build 331)** — a minor bump rolling up everything since 0.9.1: the flat
  two-tab Settings surface with project settings layered on the shared default, the simplified Add
