# Porting Synth's Claude Code Integration to OpenCode: Feasibility & Plan

> **Status (2026-07-08):** all 13 open spikes verified against a live OpenCode 1.17.7 server — **green
> light to implement end-to-end**, no blocker survived. Evidence: `opencode-spike-findings.md`.

## Executive summary

The port is not just feasible — it is a net simplification and, in most areas, an upgrade. Synth's Claude Code (CC) integration is built on three hacks that exist only because CC gives no real programmatic surface: a **PATH-shim** that shadows the `claude` binary to inject `--settings` hooks at exec, a **unix-socket** (`/tmp/synth-hook-<pid>.sock`) that `synth-hook` subprocesses write status lines into, and **PTY bracketed-paste + timed CR** to deliver text into the interactive TUI. OpenCode replaces all three with first-class primitives: an **auto-loaded plugin** (`~/.config/opencode/plugin/`) plus an **HTTP server** (`opencode serve`) exposing a typed, ordered **`/event` SSE bus** consumed via **`@opencode-ai/sdk`** (`client.event.subscribe()`), where every event natively carries a `sessionID`. Session **titles are a structured `Session.title` field** (auto-generated by OpenCode's hidden title agent) instead of `ai-title` lines scraped from transcript JSONL. MCP is **native `opencode.json`/`opencode.jsonc` config** (`mcp` key) rather than a hand-written `.mcp.json`. Message injection is a **real API** (`session.prompt` / `POST /session/:id/message`), which dissolves the paste-and-Enter timing hazard *and* the security boundary that guarded against pasting page-controlled text into a fallback shell. The genuinely hard spots are narrow and mostly about **fidelity loss, not blockers**: OpenCode's title generates **once** (no continuously-evolving title), the **`--auto` auto-approve mode suppresses `permission.asked`** so "needs-input" can vanish in unattended runs, there is **no `idle_prompt` analog**, and several event-ordering / abort-semantics details are **unverified and need a runtime spike**. No touchpoint is a true dead-end; the two "gap"-tagged items (child-session env scrub; `preferredNotifChannel` suppression) are gaps that **disappear** in OpenCode's model rather than block it.

## Architecture shift

| | Claude Code (today) | OpenCode (target) |
|---|---|---|
| **Instrumentation** | PATH-shim shadows `claude`, re-execs real binary with injected argv; must police interactive-vs-`-p` passthrough | Static plugin auto-loaded from `~/.config/opencode/plugin/` (or `.opencode/plugin/`); no binary shadowing, no argv trick |
| **Hook config** | `--settings <inline-json>` carrying a `hooks` map, deep-merged into user settings per launch | Plugin JS module on disk (`event`, `tool.execute.before/after`, `permission.ask` handlers) + per-launch `OPENCODE_CONFIG_CONTENT` env for inline config |
| **Status transport** | `synth-hook` subprocesses write JSON lines to `/tmp/synth-hook-<pid>.sock`, keyed by injected `SYNTH_SESSION_ID` | `/event` SSE stream (or in-process plugin `event` hook); every event carries native `sessionID` — correlation is intrinsic |
| **Text delivery** | Bracketed-paste + 0.35s beat + CR into the PTY TUI | `session.prompt` / `POST /session/:id/message` (structured message API); TUI becomes a viewer, not the input channel |
| **Session identity** | `--session-id <uuid>` minted by Synth, echoed on hook payloads; `--resume <id>` restores | Server mints and persists `Session.id`, read back from `session.create` / `session.created`; resume via `opencode --session <id>` / SDK fetch |
| **Titles** | `ai-title` JSONL records tailed from `transcript_path` | `Session.title` field, delivered on `session.updated` |
| **MCP** | Per-worktree `.mcp.json` (`mcpServers`, command+args) | Per-worktree `opencode.jsonc` (`mcp` key, `type:"local"`, `command` array, `environment`, `cwd`) |

**Recommended approach for Synth on OpenCode: server-first, plugin-assisted.** Synth spawns `opencode` in the PTY with a known `--port`/`--hostname` (or reads the random port) and subscribes to `/event` via the SDK for all status, title, lifecycle, and notification signals — this replaces the entire `synth-hook`/socket machinery. Text delivery (comments, feedback seeds, spawned owners) goes through `session.prompt` against the server, not the PTY. A single **global Synth plugin** is installed once to `~/.config/opencode/plugin/` for the few things the SSE stream can't do alone (per-`sessionID` browser focus via `tool.execute.before`; optional custom tools). The embedded TUI, if kept, is a **viewer** attached to the same server. This anchors liveness to the transport itself (SSE connected + non-deleted session), which is a stronger, safer "is this live right now" fact than CC's start/end pipe.

## GAPS — ranked by user impact

### 1. "Needs-input" under auto-approve — MEDIUM impact, ✅ RESOLVED by spike (was HIGH/low-confidence)
**Today:** Under `--dangerously-skip-permissions`, CC still fires PreToolUse for `AskUserQuestion`/`ExitPlanMode`, so Synth reliably shows "needs your input" even for unattended, seed/comment-driven agents.
**Verified (OpenCode 1.17.7):** the `question` channel is **independent of the permission ruleset** — `question.asked` fired even under an all-allow session. Permission auto-approve (`allow`) *does* suppress `permission.asked`, but a tool set to `ask` reliably emits `permission.asked`→`permission.replied`. So needs-input is fully covered.
**Resolution:** needs-input = `question.asked` **OR** `permission.asked`. Set `OPENCODE_ENABLE_QUESTION_TOOL=1` (the tool is env-gated) and, for unattended agents, keep the "asking" surfaces (`bash`/`edit`) at `"ask"` rather than fully allow-all — granularity CC never had. No `tool.execute.before` fallback needed. See `opencode-spike-findings.md` #1.

### 2. No continuously-evolving session title — MEDIUM impact, high confidence (well-understood loss)
**Today:** CC refines `ai-title` across the whole conversation; Synth's row title and notification identity track the drifting summary.
**Why OpenCode can't match:** OpenCode's title agent runs **once** on the first user message (`ensureTitle` early-returns unless `isDefaultTitle`) and never re-fires. Synth loses the "short evolving title"; it gets a one-shot title it can override.
**Best workaround:** Accept the one-shot title (stable notification identity is arguably better), and if evolution is wanted, ship a small plugin that calls `session.update` to regenerate on `session.idle`. **Must** configure a reachable `small_model` (e.g. a Haiku) or titles silently never generate (issue #9902).

### 3. No `idle_prompt` ambient nudge — LOW/MEDIUM impact, low confidence
**Today:** CC's Notification `idle_prompt` lets Synth nudge "the agent has been waiting on you." **Why OpenCode can't match:** `session.idle` means *turn complete*, not *user has been idle*; there is no user-idle event. **Best workaround:** no direct path today. Synth could synthesize its own idle-nudge timer client-side off `permission.asked` age if desired; the load-bearing permission/elicitation cases are fully covered by `permission.asked`.

### 4. MCP tool args carry no caller identity — MEDIUM impact, high confidence (has a clean workaround)
**Today:** the `sessionId`-everywhere browser API exists because CC gives sub-agents/tool-calls no caller identity. **Why OpenCode "can't" as a pure stdio server:** MCP has no caller-identity field in args, so a self-contained `server.mjs` still can't tell which session called. **Best workaround (an upgrade):** ship the Synth plugin that hooks `tool.execute.before`, reads `sessionID`, and sets per-session browser focus — concurrent agents auto-target their own browser and the `sessionId` param becomes optional. Keep the explicit param as a fallback for pure-MCP config. More moving parts, far cleaner semantics.

### 5. Event-ordering & abort semantics — MEDIUM impact, ✅ RESOLVED by spike
**Verified (OpenCode 1.17.7):** (a) `permission.replied` **always precedes** `session.idle` — the turn resumes (`busy`) after the reply and only later goes idle (`asked → replied → busy → idle`). (b) A user-abort emits **both** `session.error{ error.name: "MessageAbortedError" }` **and then** `session.status:idle` + `session.idle`. **Resolution:** classify `session.error` by `error.name` — `MessageAbortedError` is a clean interrupt (CC's 130/143), never an error toast; everything else is a real error. The reconcile guard is safe as-is. See `opencode-spike-findings.md` #2, #3.

### 6. `preferredNotifChannel` suppression has no analog — LOW impact, low confidence (gap that likely dissolves)
**Today:** Synth sets `notifications_disabled` so CC's own OS/terminal notifs don't double-fire. **Why OpenCode "can't":** no equivalent key documented. **But:** OpenCode has no built-in OS/terminal notification channel — native notifs exist only if a user installs a community plugin — so there's probably nothing to suppress. **Workaround:** don't ship a notify plugin; optionally disable known ones via the `plugin` config key. **Spike:** confirm no OS notification leaks.

### 7. Child-session env-scrub gap — LOW impact (a gap that disappears)
The `CLAUDE_CODE_CHILD_SESSION` unset has no analog because OpenCode's subagents are in-process child sessions distinguished by `parentID`, not env-demoted exec-nested processes. Replace the env scrub with `parentID`-based filtering of row-naming/idle events. Residual low-confidence audit: strip any `OPENCODE_*` var that could confuse a nested launch.

## Phased implementation plan

**What gets deleted wholesale:** the PATH-shim / `synth-hook` binary and its launch-interception role; the `--settings` deep-merge machinery (`buildSettingsJSON`, `takeSettingsValue`, `mergeSettings`); the `/tmp/synth-hook-<pid>.sock` transport and `HookServer` socket ingestion; `SYNTH_SESSION_ID` env injection + inheritance chain; the exec-under-`login` + spawn-as-child exit-code workaround; the transcript JSONL `ai-title` scraper; the `--session-id` minting path; the boot-and-wait poll ladder; the bracketed-paste + timed-CR delivery path. **What must be rebuilt:** an SDK/`/event` subscriber that maps OpenCode events → Synth's `SessionEvent`/`SessionStatus` model; the `Hooks.swift` `runEvent` classifier as a plugin/SDK consumer; row model keyed on `sessionID` not PTY pid; per-worktree `opencode.jsonc` writer for MCP; the browser-focus plugin.

**Phase 0 — Boot an OpenCode session end-to-end (effort: S/M).** Substitute `exec opencode` for `exec claude` (spawn-templates T1/T7); scrub `OPENCODE_*` from spawn env; strip the `claude` word from preexec, gate on `opencode`. Stand up `opencode serve` (or embedded server) + `@opencode-ai/sdk` subscription to `/event`. Deliverable: Synth launches an OpenCode row that runs interactively. Delete the availability/real-binary-resolution machinery.

**Phase 1 — Status, lifecycle, titles (effort: M, the big rewrite).** Replace `synth-hook`/socket with the SSE subscriber. Capture `sessionID` at spawn for the row↔session map (drives tap-to-jump). Configure `small_model`. Delete the exit-code socket path — classify done/error from events.

**Phase 2 — MCP / browser (effort: S/M).** Emit per-worktree `opencode.jsonc` (`mcp`, `type:"local"`, `command` array, `environment.SYNTH_WORKTREE`, `cwd`) with the existing atomic/merge-preserving write. Ship the `tool.execute.before` focus plugin so concurrent agents auto-target their browser; keep `sessionId` param as fallback.

**Phase 3 — Input-delivery, notifications, templates (effort: S/M).** Route comment/feedback/seed delivery through `session.prompt` / `POST /session/:id/message` (delete paste-and-Enter and the poll ladder); collapse liveness to "session exists on server, not errored." Wire notifications off `session.error`/`session.idle`/`permission.asked`/`permission.replied`; keep per-kind sound prefs. Config `permission` to keep `question`/plan/`edit` at `"ask"` for unattended agents (guards Gap #1). Finish template renames (`newOpencode`/`spawnOpencode`), title-frozen spawn via `POST /session {title}`, and `--session <id>` resume.
